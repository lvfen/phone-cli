## 1. 安装

```bash
# 基础安装（Android + HarmonyOS）
pip install -e "."

# 含 iOS 支持
pip install -e ".[ios]"

# 完整安装（含开发工具）
pip install -e ".[ios,dev]"
```

## 2. 环境要求

### 通用

| 依赖 | 版本要求 | 用途 |
|------|---------|------|
| Python | 3.10+ | 运行环境 |
| Pillow | ≥12.0.0 | 图像处理 |
| click | ≥8.0.0 | CLI 框架 |

### Android

| 依赖 | 版本要求 | 用途 |
|------|---------|------|
| ADB | 最新版 | 设备通信 |
| Android SDK | API 26+ | `ANDROID_HOME` 环境变量 |
| Java | 17+ | 编译 Companion APK |

```bash
# macOS 安装 ADB
brew install android-platform-tools

# 环境变量（加到 ~/.zshrc）
export ANDROID_HOME=~/Library/Android/sdk
export JAVA_HOME=/opt/homebrew/opt/openjdk@17
```

### HarmonyOS

| 依赖 | 说明 |
|------|------|
| HDC | 从 DevEco Studio 获取，需在 PATH 中 |

| 依赖 | 版本要求 | 用途 |
|------|---------|------|

| 模式 | 要求 |
|------|------|

## 3. 快速开始

### Android 一分钟上手

```bash
# 1. 确认设备连接
adb devices

# 2. 启动 daemon
phone-cli start --device-type adb

# 3. 初始化 Companion 无障碍服务（首次需要）
phone-cli companion-setup

# 4. 检查状态
phone-cli status

# 5. 开始操作
phone-cli screenshot --resize 720    # 截图
phone-cli ui-tree                     # 查看界面元素
phone-cli tap 500 500                 # 点击屏幕中心
phone-cli launch 微信                 # 打开微信

# 6. 结束
phone-cli stop
```

### iOS 一分钟上手

```bash
# 1. 检测可用运行时
phone-cli detect-runtimes --device-type ios

# 2. 启动（自动选择 / 指定运行模式）
phone-cli start --device-type ios
# 或指定：phone-cli start --device-type ios --runtime simulator --device-id <udid>

# 3. 开始操作
phone-cli launch --bundle-id com.apple.Preferences
phone-cli screenshot
phone-cli tap 500 300

# 4. 结束
phone-cli stop
```

## 4. 核心概念

### 4.1 Daemon 架构

phone-cli 采用 daemon 常驻进程模式，所有命令通过 Unix Socket 与 daemon 通信：

```
用户命令 ──> phone-cli CLI ──> Unix Socket ──> daemon 进程 ──> 设备
```

- **Per-device daemon**：每个平台一个实例（`adb` / `hdc` / `ios`），适合单 AI 会话
- **Central daemon**：`phone-cli daemon start`，统一管理多设备多会话

数据存储在 `~/.phone-cli/instances/<name>/` 下：
```
~/.phone-cli/instances/adb/
├── phone-cli.pid          # 进程 PID
├── state.json             # 状态
├── phone-cli.sock         # IPC Socket
├── logs/                  # 日志（按天滚动，30 天保留）
└── screenshots/           # 截图文件
```

### 4.2 坐标系统（0-999 相对坐标）

**所有手势命令使用 0-999 的相对坐标**，不依赖屏幕分辨率：

```
(0, 0) ───────────── (999, 0)
  │                      │
  │     (500, 500)       │     ← 屏幕正中心
  │                      │
(0, 999) ──────────── (999, 999)
```

phone-cli 自动将相对坐标映射为实际像素坐标。

> **注意**：`ui-tree` 返回的 `center_x` / `center_y` 是**像素坐标**，需要转换后才能用于 `tap` 等命令：
> ```
> 相对 X = 像素 X / 屏幕宽度 × 999
> 相对 Y = 像素 Y / 屏幕高度 × 999
> ```

### 4.3 JSON 输出

所有命令统一返回 JSON：

```json
// 成功
{"status": "ok", "data": { ... }}

// 失败
{"status": "error", "error_code": "ERROR_CODE", "error_msg": "描述"}
```

### 4.4 应用名称映射

支持用**中文名**直接启动常用 App，内置 300+ 应用映射：

```bash
phone-cli launch 微信       # → com.tencent.mm
phone-cli launch 淘宝       # → com.taobao.taobao
phone-cli launch 抖音       # → com.ss.android.ugc.aweme
phone-cli launch 设置       # → com.android.settings（Android）
```

## 5. 命令详解

### 5.1 Daemon 管理

#### `start` — 启动 daemon

```bash
phone-cli start --device-type adb          # Android
phone-cli start --device-type hdc          # HarmonyOS
phone-cli start --device-type ios          # iOS（自动选择运行时）
phone-cli start --device-type ios --runtime device --device-id <udid>
phone-cli start --device-type ios --runtime simulator --device-id <sim_udid>
phone-cli start --device-type ios --runtime app-on-mac
```

| 选项 | 说明 |
|------|------|
| `--device-type adb\|hdc\|ios` | 平台类型 |
| `--device-id TEXT` | 指定设备 ID |
| `--runtime device\|simulator\|app-on-mac` | iOS 运行模式 |

#### `stop` — 停止 daemon

```bash
phone-cli stop              # 停止当前实例
phone-cli stop --all        # 停止所有实例
```

#### `status` — 查看状态

```bash
phone-cli status
```

返回值包含：`device_type`、`device_status`、`companion_status`（Android）、`screen_size` 等。

#### `daemon` — Central daemon 管理

```bash
phone-cli daemon start      # 启动中央 daemon
phone-cli daemon status     # 查看状态
phone-cli daemon stop       # 停止
```

### 5.2 设备管理

#### `devices` — 列出已连接设备

```bash
phone-cli devices
```

#### `set-device` — 切换目标设备

```bash
phone-cli set-device <DEVICE_ID>
```

多设备场景下，先 `devices` 查看列表，再 `set-device` 绑定。

#### `device-info` — 设备详情

```bash
phone-cli device-info
```

返回型号、分辨率、系统版本等信息。

#### `detect-runtimes` — 检测 iOS 运行时

```bash
phone-cli detect-runtimes --device-type ios
```

自动发现可用的 iOS 真机、模拟器、App on Mac 候选项。

### 5.3 手势操作

> Android 下所有手势命令**默认通过 Companion 无障碍服务**执行，返回值含 `"source"` 字段。

#### `tap` — 单击

```bash
phone-cli tap <X> <Y>                    # 默认 companion 优先
phone-cli tap 500 300                     # 点击屏幕上方中间
phone-cli tap 500 300 --type adb          # 强制 ADB
phone-cli tap 500 300 --type companion    # 强制 companion
```

#### `double-tap` — 双击

```bash
phone-cli double-tap <X> <Y>
phone-cli double-tap 500 500
```

#### `long-press` — 长按

```bash
phone-cli long-press <X> <Y>
phone-cli long-press 500 500
```

#### `swipe` — 滑动

```bash
phone-cli swipe <X1> <Y1> <X2> <Y2>
phone-cli swipe 500 800 500 200          # 从下往上滑（上滑）
phone-cli swipe 500 200 500 800          # 从上往下滑（下拉）
phone-cli swipe 800 500 200 500          # 从右往左滑
```

#### `type` — 输入文本

```bash
phone-cli type "Hello World"              # 输入英文
phone-cli type "你好世界"                  # 输入中文
```

会自动清除当前输入框内容后输入新文本。

#### `back` — 返回键

```bash
phone-cli back
```

#### `home` — Home 键

```bash
phone-cli home
```

### 5.4 页面观察

#### `ui-tree` — 获取 UI 元素树

```bash
phone-cli ui-tree
```

返回页面所有可见元素的结构化数据：

```json
{
  "status": "ok",
  "data": {
    "source": "companion",           // companion 或 uiautomator
    "elements": [
      {
        "text": "微信",
        "resource_id": "com.tencent.mm:id/title",
        "class": "android.widget.TextView",
        "center_x": 720,             // 像素坐标
        "center_y": 224,
        "clickable": true,
        "scrollable": false,
        "editable": false,
        "content_description": "",
        "node_id": "0.0.1.2"
      }
    ]
  }
}
```

**关键字段：**
- `source` — 数据来源（`companion` / `uiautomator`）
- `text` — 元素文本
- `center_x`, `center_y` — 元素中心的**像素坐标**
- `clickable` — 是否可点击
- `editable` — 是否可编辑（输入框）
- `node_id` — 节点 ID（可用于 `click-node`）

#### `find-nodes` — 按条件搜索节点（Android）

```bash
phone-cli find-nodes --text "确定"                    # 精确匹配文本
phone-cli find-nodes --text-contains "设置"            # 模糊匹配
phone-cli find-nodes --resource-id "com.example:id/btn" # 按 resource_id
phone-cli find-nodes --class-name "android.widget.Button" --clickable  # 按类名+可点击
```

| 选项 | 说明 |
|------|------|
| `--text TEXT` | 精确匹配文本 |
| `--text-contains TEXT` | 部分匹配文本 |
| `--resource-id TEXT` | 匹配 resource_id |
| `--class-name TEXT` | 匹配 class 名 |
| `--clickable` | 仅返回可点击节点 |

#### `click-node` — 按节点 ID 点击（Android）

```bash
phone-cli click-node "0.0.1.2"
phone-cli click-node "0.0.1.2" --fallback-x 500 --fallback-y 300
```

通过 `ui-tree` 返回的 `node_id` 直接点击元素，比坐标点击更精确。

#### `screen-context` — 获取可交互元素摘要（Android）

```bash
phone-cli screen-context
```

返回当前页面的可交互元素精简列表，比 `ui-tree` 更简洁。

#### `search-click` — 搜索并点击（Android，一步完成）

```bash
phone-cli search-click --text "确定"
phone-cli search-click --text-contains "下一步" --clickable
phone-cli search-click --resource-id "com.example:id/submit" --index 0
```

将搜索 + 点击合并为 Companion 一次往返，减少延迟。

| 选项 | 说明 |
|------|------|
| `--text TEXT` | 精确匹配 |
| `--text-contains TEXT` | 模糊匹配 |
| `--resource-id TEXT` | resource_id 匹配 |
| `--class-name TEXT` | class 名匹配 |
| `--package-name TEXT` | 包名匹配 |
| `--clickable` | 仅可点击节点 |
| `--index INTEGER` | 匹配到多个时选第几个（默认 0） |

#### `search-set-text` — 搜索输入框并填入文本（Android，一步完成）

```bash
phone-cli search-set-text "Hello" --resource-id "com.example:id/input"
phone-cli search-set-text "搜索内容" --match-text "请输入关键字"
phone-cli search-set-text "用户名" --text-contains "请输入"
```

| 选项 | 说明 |
|------|------|
| `TEXT`（必填） | 要输入的文本 |
| `--match-text TEXT` | 精确匹配目标输入框文本 |
| `--text-contains TEXT` | 模糊匹配目标输入框 |
| `--resource-id TEXT` | resource_id 匹配 |
| `--class-name TEXT` | class 名匹配 |
| `--index INTEGER` | 匹配到多个时选第几个 |
| `--no-focused-fallback` | 禁用回退到当前焦点输入框 |

### 5.5 应用管理

#### `launch` — 启动 App

```bash
# 按中文名启动（内置 300+ 应用映射）
phone-cli launch 微信
phone-cli launch 设置
phone-cli launch 淘宝

# 按包名 / bundle ID 启动
phone-cli launch --bundle-id com.tencent.mm
phone-cli launch --bundle-id com.apple.Preferences

# 按 .app 路径启动（iOS App on Mac）
phone-cli launch --app-path "/path/to/Demo.app"
```

#### `get-current-app` — 获取前台应用

```bash
phone-cli get-current-app
```

返回当前前台应用的包名/bundle ID。

#### `app-state` — 查询应用状态

```bash
phone-cli app-state --package com.tencent.mm          # Android
phone-cli app-state --bundle-id com.apple.Preferences  # iOS
```

#### `wait-for-app` — 等待应用就绪

```bash
phone-cli wait-for-app com.tencent.mm --timeout 10
phone-cli wait-for-app --bundle-id com.apple.Preferences --timeout 10
phone-cli wait-for-app com.example.app --state running --timeout 15
```

| 选项 | 说明 |
|------|------|
| `PACKAGE`（可选） | Android 包名 |
| `--bundle-id TEXT` | iOS bundle ID |
| `--timeout INTEGER` | 最大等待秒数 |
| `--state resumed\|running` | 目标状态（默认 `resumed`） |

#### `install` — 安装 APK（Android）

```bash
phone-cli install /path/to/app.apk
phone-cli install /path/to/app.apk --launch    # 安装后自动启动
```

### 5.6 截图与屏幕检测

#### `screenshot` — 截图

```bash
phone-cli screenshot                        # 原始分辨率
phone-cli screenshot --resize 720           # 缩放到宽 720px（推荐）
phone-cli screenshot --task-id "test1" --step 3  # 带任务标记
```

| 选项 | 说明 |
|------|------|
| `--resize INTEGER` | 缩放宽度（像素），高度等比缩放 |
| `--task-id TEXT` | 任务 ID（影响文件命名） |
| `--step INTEGER` | 步骤序号 |

返回截图文件路径和尺寸。

#### `check-screen` — 屏幕健康检查

```bash
phone-cli check-screen
phone-cli check-screen --threshold 0.95
```

检测屏幕是否全黑/全白（设备异常），返回 `all_black` / `all_white` / `normal`。

### 5.7 Android Companion（无障碍服务）

Companion 是 phone-cli 的 Android 辅助应用，通过无障碍服务提供：
- **更快更精确的手势执行**（vs ADB shell input）
- **高质量 UI 元素树**（vs uiautomator dump）
- **节点搜索与点击**（find-nodes、search-click 等高级命令）

#### `companion-setup` — 初始化 Companion

```bash
phone-cli companion-setup
```

自动完成：检查 APK → 编译（首次约 1-3 分钟）→ 安装到设备 → 启用无障碍权限 → 建立端口转发。

#### `companion-status` — 查看 Companion 状态

```bash
phone-cli companion-status
```

返回详细的服务健康信息。

#### `companion-preflight` — 预检诊断

```bash
phone-cli companion-preflight
```

检查 Companion 的每个环节，返回可操作的诊断建议。

## 6. 平台指南

### 6.1 Android

#### 完整工作流

```bash
# 1. 检查设备
adb devices

# 2. 启动 daemon + Companion
phone-cli start --device-type adb
phone-cli companion-setup

# 3. 确认状态
phone-cli status                # 确认 companion_status: ready

# 4. 操作设备
phone-cli ui-tree               # 观察页面
phone-cli tap 500 300           # 点击
phone-cli screenshot --resize 720  # 截图验证

# 5. 使用高级 Companion 命令
phone-cli find-nodes --text "确定"
phone-cli search-click --text "确定"
phone-cli search-set-text "Hello" --resource-id "com.example:id/input"

# 6. 结束
phone-cli stop
```

#### Companion 优先级规则

| 操作 | 优先级 1 | 优先级 2 | 优先级 3 |
|------|---------|---------|---------|
| 手势 | Companion | ADB shell (`--type adb`) | — |
| 页面观察 | Companion ui-tree | uiautomator | 截图 |

返回值的 `source` 字段告诉你实际走了哪个通道。

#### 多设备

```bash
phone-cli devices                # 列出所有设备
phone-cli set-device <ID>        # 切换到指定设备
```

### 6.2 HarmonyOS

```bash
# 1. 检查设备
hdc list targets

# 2. 启动
phone-cli start --device-type hdc

# 3. 操作（命令与 Android 一致）
phone-cli ui-tree
phone-cli tap 500 500
phone-cli screenshot --resize 720

# 4. 结束
phone-cli stop
```

> HarmonyOS 不支持 Companion，手势通过 HDC uitest 执行。

### 6.3 iOS

#### 检测可用运行时

```bash
phone-cli detect-runtimes --device-type ios
```

返回所有可用候选项（真机 / 模拟器 / App on Mac）。

#### 真机

```bash
phone-cli start --device-type ios --runtime device --device-id <udid>
phone-cli launch --bundle-id com.tencent.xin
phone-cli wait-for-app --bundle-id com.tencent.xin --timeout 10
phone-cli ui-tree        # WDA 提供完整 UI 树
phone-cli screenshot
```

#### 模拟器

```bash
# 先启动模拟器
xcrun simctl boot <sim_udid>

phone-cli start --device-type ios --runtime simulator --device-id <sim_udid>
phone-cli launch --bundle-id com.apple.Preferences
phone-cli screenshot     # 模拟器 ui-tree 暂不可用，用截图
```

#### App on Mac（Apple Silicon）

```bash
phone-cli start --device-type ios --runtime app-on-mac
phone-cli launch --app-path "/path/to/Demo.app"
phone-cli ui-tree        # AX 树质量取决于 App 实现
phone-cli screenshot
```

#### iOS 自动选择规则

`phone-cli start --device-type ios`（不带 `--runtime`）的行为：

| 候选数 | 行为 |
|--------|------|
| 0 | 报错 `NO_AVAILABLE_IOS_RUNTIME` |
| 1 | 自动选择并启动 |
| 2+ | 报错 `RUNTIME_SELECTION_REQUIRED`，需手动指定 |

## 7. 多实例并行

phone-cli 支持 Android / HarmonyOS / iOS daemon 同时运行：

```bash
# 分别启动
phone-cli --instance adb start --device-type adb
phone-cli --instance ios start --device-type ios --runtime simulator

# 分别操作（用 --instance 指定）
phone-cli --instance adb screenshot
phone-cli --instance ios screenshot
phone-cli --instance adb tap 500 500
phone-cli --instance ios tap 500 500

# 查看各自状态
phone-cli --instance adb status
phone-cli --instance ios status

# 停止
phone-cli --instance ios stop       # 只停一个
phone-cli stop --all                # 全部停止
```

> 如果只有一个实例在运行，可以省略 `--instance`。

## 8. 坐标系统

### 相对坐标（0-999）

所有手势命令（`tap`、`swipe` 等）使用 **0-999 的相对坐标**：

| 位置 | 坐标 |
|------|------|
| 左上角 | (0, 0) |
| 右上角 | (999, 0) |
| 正中心 | (500, 500) |
| 左下角 | (0, 999) |
| 右下角 | (999, 999) |

### 像素坐标 → 相对坐标转换

`ui-tree` 返回的是像素坐标，使用前需要转换：

```python
# 假设屏幕分辨率 1440×3200，元素像素坐标 (720, 1600)
relative_x = int(720 / 1440 * 999)    # → 499
relative_y = int(1600 / 3200 * 999)   # → 499
# phone-cli tap 499 499
```

可以通过 `phone-cli status` 获取屏幕分辨率（`screen_size` 字段）。

### 常用滑动手势

```bash
phone-cli swipe 500 700 500 300       # 上滑（刷新/翻页）
phone-cli swipe 500 300 500 700       # 下拉（刷新/展开通知栏）
phone-cli swipe 800 500 200 500       # 左滑
phone-cli swipe 200 500 800 500       # 右滑
```

## 9. 输出格式

所有命令返回标准 JSON，便于脚本解析：

### 成功

```json
{"status": "ok", "data": {"source": "companion", "x": 720, "y": 1600}}
```

### 失败

```json
{"status": "error", "error_code": "APP_NOT_FOUND", "error_msg": "App not found: xxx"}
```

### 常见 error_code

| 错误码 | 说明 |
|--------|------|
| `DAEMON_NOT_RUNNING` | Daemon 未启动 |
| `DEVICE_NOT_CONNECTED` | 设备未连接 |
| `APP_NOT_FOUND` | 应用未找到 |
| `UI_TREE_UNAVAILABLE` | 无法获取 UI 树 |
| `COMPANION_NOT_READY` | Companion 服务未就绪 |
| `COMPANION_BUILD_FAILED` | Companion APK 编译失败 |
| `NO_AVAILABLE_IOS_RUNTIME` | 没有可用的 iOS 运行时 |
| `RUNTIME_SELECTION_REQUIRED` | 有多个运行时，需手动选择 |
| `SCREENSHOT_FAILED` | 截图失败 |
| `TIMEOUT` | 操作超时 |

### 用 jq 解析输出

```bash
# 获取 source 字段
phone-cli tap 500 500 | jq -r '.data.source'

# 获取所有有文本的 UI 元素
phone-cli ui-tree | jq '[.data.elements[] | select(.text != "") | {text, center_x, center_y, clickable}]'

# 获取当前前台应用包名
phone-cli get-current-app | jq -r '.data.package'
```

## 10. 常见问题

### Daemon 相关

| 问题 | 解决方案 |
|------|---------|
| `DAEMON_NOT_RUNNING` | `phone-cli start --device-type adb` |
| daemon 无响应 | `phone-cli stop` 后重新 `start` |
| 端口被占用 | `lsof -i :5037` 查看占用进程并 kill |

### Android 相关

| 问题 | 解决方案 |
|------|---------|
| `adb devices` 无设备 | 检查 USB 连接，开启 USB 调试 |
| Companion 编译失败 | 检查 `ANDROID_HOME` 和 Java 17+，查看 `error_msg` |
| Companion 不可用 | 重新运行 `phone-cli companion-setup` |
| 截图全黑 | 模拟器不要用 `-no-window` 启动，尝试重启 |
| ui-tree 节点质量低 | Flutter/游戏/WebView 页面正常现象，用截图替代 |
| ADB server 冲突 | `adb kill-server && adb start-server` |

### iOS 相关

| 问题 | 解决方案 |
|------|---------|
| 检测不到设备 | 确认 USB 连接并信任，`tidevice list` 验证 |
| WDA 启动失败 | 重新编译部署 WebDriverAgent |
| 模拟器 ui-tree 不可用 | MVP 限制，使用截图替代 |
| App on Mac 权限不足 | 系统设置 → 隐私 → 辅助功能 + 录屏 |

### 模拟器相关

| 问题 | 解决方案 |
|------|---------|
| `qemu-system not found` | 先 `cd $ANDROID_HOME/emulator` 再执行 |
| snapshot 恢复后黑屏 | 用 `-no-snapshot-load` 冷启动 |
| 模拟器存在但 ADB 无设备 | `lsof -i :5037` → kill → `adb kill-server && adb start-server` |
